# AI 业务标注生成指南

AI 在 Step 3 阶段分析用户提供的 Markdown 源文档，生成四项业务标注。所有标注必须有据可查，不可凭空推断。

## 四项标注字段

### 1. 文档用途和相关注释信息

**目标**: 概括文档的核心目的和应用方向。

**生成规则**:
- 长度: 2-3 句话，不超过 100 字
- 回答"这份文档是做什么的"和"谁在什么情况下需要它"
- 从文档标题、引言段落、总结段落中提取核心信息
- 使用业务化语言而非技术描述

**提取策略**:
1. 读取文档标题（一级标题）和第一段
2. 读取文档结尾总结（如有）
3. 综合生成用途描述

**示例**:

源文档标题: "区域销售团队健康产品培训手册"

```
该文档用于指导区域销售团队了解健康产品线的核心卖点和销售话术，
帮助一线销售人员在客户沟通中准确传达产品价值。
```

### 2. 关键内容关联词汇

**目标**: 提取文档中的核心术语和关键概念。

**生成规则**:
- 数量: 5-15 个关键词/短语
- 每个词汇必须标注其在源文档中的出处（段落编号或章节名）
- 优先提取: 专有名词 > 行业术语 > 关键动作/流程名
- 排除: 通用词汇（如"管理""工作""文档"等）

**提取策略**:
1. 扫描所有章节标题，提取专有名词
2. 统计全文高频词汇（排除停用词），取 top 结果
3. 识别加粗、斜体或引用标记的重点内容
4. 每个词汇回溯到源文档的具体位置

**输出格式**:

```
健康管理(第2章「产品概述」), 产品培训(第5章「培训流程」),
销售话术(第8章「客户沟通」), BMI指数(第3章「核心指标」),
客户画像(第6章「目标客户」)
```

**禁止**: 生成源文档中未出现的词汇。如果某个词是合理推断但未直接出现在文档中，用方括号标记: `[推断] 词汇`。

### 3. 适用场景

**目标**: 推断该文档在什么业务场景下被使用。

**生成规则**:
- 数量: 2-5 个场景
- 每个场景用简短短语描述（5-15 字）
- 场景应具体可行动，不要过于抽象

**提取策略**:
1. 从文档内容推断其使用场景
2. 考虑用户提供的部门/科室信息，结合岗位职能
3. 如果文档中明确提到使用场景，优先直接引用

**好的场景描述**:
- "新员工入职培训"
- "季度产品更新宣讲"
- "客户拜访前的产品准备"
- "区域月度销售会议"

**差的场景描述**（过于抽象）:
- "日常工作"
- "业务开展"
- "知识学习"

### 4. 适用人群

**目标**: 推断该文档面向哪些角色或岗位。

**生成规则**:
- 数量: 1-5 个角色/岗位
- 使用具体岗位名称而非泛称
- 如果文档明确提到受众，直接引用

**提取策略**:
1. 检查文档是否明确提到目标受众
2. 从内容的专业深度推断面向的角色级别
3. 结合部门/科室信息推断岗位范围

**好的人群描述**:
- "一线销售人员"
- "区域经理"
- "产品培训师"
- "新入职员工"

**差的人群描述**:
- "所有人"
- "相关人员"
- "需要的人"

## 标注生成完整示例

**源文档**: `product-training-guide.md`（健康产品培训手册，约3000字）

```
【AI 业务标注】

1. 文档用途:
   该文档是面向一线销售团队的健康产品培训手册，系统介绍了
   Wellness 产品线的核心功能、竞品对比和标准销售话术，
   帮助销售人员在客户沟通中准确传达产品价值。

2. 关键内容关联词汇:
   Wellness产品线(第1章「产品概述」),
   BMI健康指数(第2章「核心指标」),
   客户需求分析(第3章「客户画像」),
   竞品对比矩阵(第4章「市场分析」),
   标准销售话术(第5章「销售工具」),
   客户异议处理(第5章「销售工具」),
   健康管理方案(第6章「解决方案」),
   续费转化(第7章「客户维护」),
   NPS评分(第8章「效果追踪」)

3. 适用场景:
   - 新员工入职产品培训
   - 季度产品知识更新宣讲
   - 客户拜访前的产品准备
   - 销售团队月度技能复盘
   - 竞品信息更新学习

4. 适用人群:
   - 一线销售人员
   - 区域销售经理
   - 产品培训师
   - 新入职销售员工
```

## 特殊情况处理

### 文档过短（< 200 字）
- 关键词汇数量可降至 3-5 个
- 适用场景可降至 1-2 个
- 在标注中注明"文档内容较短，标注仅基于有限信息"

### 文档内容模糊
- 无法准确推断时，在对应字段标注 `[待确认]`
- 向用户说明原因，请求补充信息

### 文档包含多个主题
- 标注中应反映多个主题
- 关键词汇按主题分组标注
- 建议用户考虑拆分为多个文档
